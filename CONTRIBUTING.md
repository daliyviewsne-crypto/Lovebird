# Love Birds - Contributing Guide

Thank you for your interest in contributing to Love Birds! We're excited to have you.

## Code Style

- Use TypeScript for type safety
- Follow ESLint/Prettier formatting
- 2-space indentation
- Meaningful variable/function names

## Git Workflow

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Commit changes: `git commit -am 'Add feature'`
4. Push to branch: `git push origin feature/your-feature`
5. Submit pull request

## Pull Request Process

1. Update README.md with any new functionality
2. Add tests for new features
3. Ensure all tests pass
4. Request review from maintainers

## Reporting Issues

- Check existing issues first
- Provide clear bug reproduction steps
- Include environment details
- Attach screenshots/logs if applicable

## Questions?

Open a GitHub Discussion or contact the maintainers.
